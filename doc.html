Overview

Livescribe captures live speech, transcribes it, and builds an animated knowledge graph as people talk. All processing happens locally — no data leaves the system.

There are three views, each for a different moment:

Display
Projected for the audience. Shows the live knowledge graph and transcript. No controls.
/
Monitor
The operator's control surface. Session setup, audio capture, live metrics.
/monitor
Sessions
Post-event review. Browse sessions, generate recaps, export, synthesize.
/sessions

The display is purely passive — it never creates or ends sessions. The monitor owns the session lifecycle. The sessions page is for after the event.

Before a Session

Setup checklist

  1. Open /monitor on the operator's laptop
  2. Open / on the audience-facing screen (projector, TV, shared screen)
  3. The display will show the Livescribe logo and "Waiting for session to start"

Session setup (Monitor)

The monitor starts in the setup phase. Fill in:

FieldNotes
TopicThe session name. Shows in the display header and recap. Be descriptive — it helps the AI focus.
LanguageSet the primary language being spoken. Auto-detect works for English. For Norwegian, Swedish, or mixed sessions: set explicitly for best results.
Audio DeviceSelect the microphone. The system remembers your last choice.

Click Start Session. This creates the session on the server, starts audio capture, and switches to the live monitoring view. The display will wake up and show the graph.

Tip: Test audio before the meeting starts. The level meter should react when you speak. If it doesn't, check your microphone selection and browser permissions.

Language notes

  • English: Auto-detect works well
  • Norwegian: Set language to Norwegian explicitly. Auto-detect is unreliable for Norwegian.
  • Swedish: Works well when set explicitly
  • Mixed sessions: Set to the primary language. The STT will still catch code-switching to English.

During a Session

Monitor layout

The live monitor has three columns:

ColumnWhat's there
LeftAudio level meter, VAD indicator (speaking/silent), waveform, language picker, Start/Stop/Mute buttons
CenterConnection health (WS/STT/LLM), session stats (segments, chars, detected language), metrics grid, STT and LLM backend switchers
RightGraph mirror — a small replica of what the audience sees

The transcript bar at the bottom shows the live transcript as it comes in.

Audio controls

ButtonWhat it does
Start / StopStart or stop audio capture. When stopped, no audio is sent to the server.
MuteAudio is still captured (you can see the meter move) but not sent. Useful for a quick private aside.

Status indicators

DotMeaning
GreenHealthy — connected and processing
AmberWarning — rate limited or circuit breaker active
RedError — disconnected or failing

What to watch

  • STT Latency: How long each audio chunk takes to transcribe. Under 2s is good. Over 5s means the STT is struggling.
  • LLM Latency: How long graph analysis takes. Under 20s is normal. Over 40s means the system is under load — notify the system administrator or switch to Gemini Flash (cloud) as fallback.
  • Circuit Breaker: Should say "closed". If it says "open", the LLM is failing and the system is backing off. Transcript keeps flowing, but the graph won't update until it recovers.
  • Graph (N / 24): How many nodes are active vs the 24-node budget. A full graph is fine — the reconciler manages it automatically.

Switching backends live

You can change STT backend and LLM during a session without stopping. Use this if:

  • The STT is struggling with the language — switch backends
  • The LLM is slow or erroring — switch from local (Gemma) to cloud (Gemini Flash) as fallback

Ending a session

Click End Session in the header bar. This:

  1. Stops audio capture
  2. Stores a final graph snapshot
  3. Saves the session summary
  4. Returns the monitor to the setup phase
  5. The display goes back to "Waiting for session to start"

All data is preserved. You can start a new session immediately.

Display View

The display (/) is what the audience sees. It's completely passive — the operator controls everything from the monitor.

What's on screen

  • Knowledge graph: Animated force-directed graph. Nodes are concepts, edges are relationships. Colours group related ideas.
  • Transcript: Live scrolling text in the sidebar. Latest chunk is highlighted, older text fades.
  • Status pill: Top-left corner shows the current state
LEDMeaning
Green pulseListening — audio is being captured and transcribed
Purple pulseMapping — AI is analysing the transcript
Amber pulseDegraded — LLM issue, transcript still flowing

Graph interactions

The display supports direct interaction (useful if the operator has access to the projected screen):

ActionHow
Drag nodesClick and drag to reposition
Pin / UnpinRight-click → Pin. Pinned nodes never get automatically removed.
PromoteRight-click → Promote. Boosts a node's importance score.
RenameRight-click → Rename. Fix a label the AI got wrong.
MergeRight-click → Merge into... then click the target node. Combines duplicates.
HideRight-click → Hide. Removes a node from the graph.

Keyboard shortcuts

KeyAction
BToggle big-screen mode (larger fonts, thicker strokes — optimised for projection)

How the graph stays clean

  • Budget: Max 24 active nodes. New concepts push out the least important ones.
  • Decay: Nodes not mentioned for 12 minutes are automatically parked (removed from view but remembered).
  • Reactivation: Parked concepts come back if mentioned 2+ times within 3 minutes.
  • Scoring: Each node has an importance score: 45% recency, 35% frequency, 20% connectedness. Pinned nodes get a bonus.

After a Session

Go to /sessions to work with completed sessions. The monitor setup page also has a "Browse past sessions" link.

Session list

Shows all active sessions with topic, date, duration, segment count, and recap status. Click any row to open the detail view.

Session detail — three tabs

TabWhat's there
RecapAI-generated session recap. If none exists, click "Generate Recap" to create one. See below for what's in a recap.
TranscriptFull transcript with timestamps and STT metadata (detected language, backend, latency per segment).
Mind MapGraph evolution playback. Step through snapshots to see how the graph developed over time.

Recap structure (v2)

The recap is designed to surface insight, not produce meeting minutes:

SectionWhat it contains
Elevator Pitch2-3 sentences a participant could say out loud about what the session means. Written in the session's language.
Three Things to RetainThe three ideas that should survive the week. Forces prioritisation.
Non-Obvious ConnectionsLinks between topics that participants likely didn't notice. Drawn from the knowledge graph edges. 0-3 items — never fabricated.
ContradictionsWhere the discussion diverged from stated positions. Often empty — that's fine.
SummaryOne paragraph reference summary (collapsed by default).
Decisions / Open ThreadsWhat was decided and what's still unresolved (collapsed by default).
Tip: The recap is generated in the session's language. If the session was in Norwegian, the recap will be in Norwegian.

Transcript cleaning

The "Clean Transcript" button in the transcript tab sends the raw STT output through the LLM to fix:

  • Misspelled words and garbled text
  • Missing punctuation and capitalisation
  • STT hallucination loops (repeated words/phrases)
  • Inconsistent name spellings

After cleaning, changed segments are highlighted with a green border. Toggle "Show raw STT" to compare. The recap uses cleaned text when available.

Exporting

Each tab has its own export options:

TabExport options
RecapExport PDF, Copy Markdown, Download .md
TranscriptExport PDF, Copy Markdown, Download .md (respects clean/raw toggle)

PDF export opens a print-ready document in a new tab. Click "Save as PDF" or use Ctrl+P / Cmd+P.

Cross-session synthesis

When you've run multiple sessions in a day (e.g. separate agenda items), you can synthesise across them:

  1. Select 2+ sessions using the checkboxes in the session list
  2. Click Synthesize in the action bar
  3. Each session must have a recap generated first (the synthesis uses recaps + knowledge graphs as input)
  4. The result shows connections across sessions, how ideas evolved, and tensions between sessions

Archiving

Select sessions and click Archive to move them out of the active list. Archived sessions are accessible at /sessions/archive and can be unarchived at any time. No data is deleted.

Troubleshooting

No audio in the monitor

  • Check browser permissions — the monitor needs microphone access
  • Make sure you selected a device in the audio device picker
  • Try a different browser (Chrome recommended)
  • The level meter should move when you speak. If it doesn't, the device isn't capturing.

Transcript appears but graph doesn't update

  • Check the LLM status dot in the monitor — amber means the circuit breaker is active
  • The graph needs at least 50 characters of new text and waits 20 seconds between analyses
  • Try switching to Gemini Flash (cloud) as LLM fallback in the monitor
  • If latency is consistently over 40s, notify the system administrator

STT quality is poor

  • Set the language explicitly rather than using auto-detect
  • Switch STT backends in the monitor — different backends handle different languages better
  • Use the "Clean Transcript" feature post-session to fix errors

High LLM latency

  • Switch to Gemini Flash (cloud) as LLM fallback — use the LLM picker in the monitor
  • If the problem persists, contact the system administrator — the server may need services rebalanced

Display shows "Waiting for session"

  • The display is passive — it waits for the monitor to start a session
  • Check that the monitor is connected (WS dot should be green)
  • If you refresh the display during a session, it will auto-reconnect and restore the graph

WebSocket disconnects

  • Both monitor and display auto-reconnect automatically
  • The display will restore the current session state on reconnect
  • If disconnects persist, check your network connection or contact the system administrator

Data management

To keep the session list clean, use the Archive feature to move old sessions out of the active list. Archived sessions can be unarchived at any time. If you need a full database reset, contact the system administrator.
